# Evolution
Creating a game to match and evolve characters
**Step 1**
  Creating a function that allows the user to click on the eggs. 

**Step 2**
  The game needs to have a randomizer that shows different eggs for every click the user makes. 
  Every match leads to another event, it creates new monsters with different appeals.

**Final Step UI part 1**

  **Final Step UI part 2**

**RULES OF THE GAME**
  **Approach #1**
  The player needs to combine as many eggs as possible within a time frame (Is not confirmed)
  Every egg is randomized therefore it might delay the player to make certain combinations. For every combination the player earns "x" amount of points. There is   no win condition the game is aimed at creating records, for ex: Player makes 10k points first game, 2nd game he makes 20k that sets the new record. A reward      system might be implemented later, that provides players with items, for ex: A golden egg that allows the player to make any combination, a timer egg that      extends the duration of the clock, etc.

  **Approach #2**
    The game is more progressive-like (becomes a longer game) the user has a set environment, for example a jungle, certain types of eggs can only be seen there and the eggs the player encounters are also randomized with the possibility of them being repeated since they only appear in that enviroment. A reward system in this approach would use the point system which allows that for certain milestones the user reaches a new enviroment is unlocked. The endgoal in this approach is to unlock all the eggs the game has to offer.

**THINGS TO CONSIDER**
  Approach 1 takes less time and its faster but shorter. Approach 2 takes more time but opens the doors to many additions to the game and more entertaining in the long run.
